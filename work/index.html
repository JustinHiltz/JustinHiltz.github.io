JustinHiltz.com

When I redid this site I wanted it to be simple and straightforward to update, use a responsive front-end framework, and, as with anything at this stage of my development career, I wanted to learn something new.

I had heard of GitHub Pages and the more I looked into it I found it was the perfect fit for the intentions of the project. I can update the site using a simple $ git push. The site is hosted for free by GitHub and I can use my own domain name. Awesome.

For the content of the site I settled on Jekyll which is a static website generator written in Ruby. It has a little web server in it so I can instantly preview my changes before pushing them.

Foundation is the front-end framework I chose to use and I have been very happy with it. The grid is easy to use and the styling is clean and minimal.

Over the course of building this site I've learned how to use multiple technologies to work together to make something cool. Jekyll uses layout templates and partials so getting that to work nicely with Foundation took some care to prevent duplicate code.

Department 56 Store

This is a project I had some fun making. It's the first Rails app I've made completely on my own (not following a tutorial). It features a catalogue with filters for each collection of Department 56 collectibles, a simple shopping cart, and order placement.

The functionality of the site is all written in Ruby, the database is sqlite3, and the front-end framework is Foundation.
